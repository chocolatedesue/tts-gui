# ADR-001: UI 架构重设计 — 关注点分离

## Status: Accepted

## Context

当前 UI 将生成参数、播放控制、状态信息混在同一页面，导致：
- 视觉密度不均，观感不一致
- 播放和生成概念耦合
- 状态反馈（左下角小字）不够明显
- 侧栏图标过小，辨识度低

## Decision

### 1. 播放器与生成分离

- **生成**是"制造音频"的动作，参数（语速/音调/音量）决定产出物本身
- **播放**是"消费音频"的动作，控制（音量/进度/倍速）是运行时实时调节
- 生成参数移入 Settings 页；播放器做成底部固定工具栏

### 2. 三区布局

```
┌─────────┬────────────────────────────────────┐
│         │                                    │
│  侧栏    │         内容区（合成/设置）           │
│  120px  │                                    │
│  大图标  │                                    │
│  +文字   │                                    │
│         │                                    │
├─────────┼────────────────────────────────────┤
│         │  ▶ ■ ──●─────── 1:23/3:45  🔊━━ 1x │
└─────────┴────────────────────────────────────┘
```

- **侧栏** (120px): 大图标 + 文字，页面导航
- **内容区**: 合成页 / 设置页
- **底部播放栏**: 始终可见（无音频时灰显）

### 3. 合成页精简

合成页只保留：
- 左右分栏文本区（原文 | 清洗后）+ 清洗开关
- 语音选择（语言/性别/音色 + 试听）
- 生成按钮

### 4. 底部播放栏控件

- 播放/暂停
- 停止
- 进度条 (可拖动 seek)
- 当前时间 / 总时长
- 音量滑条（播放音量）
- 播放速率 (1x/1.5x/2x)

### 5. 状态通知

- 移除固定 status 文字
- 改为应用内 Toast 卡片（窗口右上角，自动消失）
- 成功/进行中/错误用不同颜色区分

### 6. 窗口缩放

- 窗口支持自由拖动缩放
- 所有内容区域自适应布局（stretch）

## Consequences

- 合成页更干净，只关注"输入 → 生成"
- 播放体验独立，类似音乐播放器
- 状态变化更明显（Toast 比小字更醒目）
- Settings 页承载更多配置（语音预设 + LLM + 音频设备 + TTS 参数）
